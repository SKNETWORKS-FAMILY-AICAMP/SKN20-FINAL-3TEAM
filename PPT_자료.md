## 슬라이드 1: 프로젝트 소개

**프로젝트명**: ARAE (아래)
**한줄 소개**: 건축 평면도 AI 분석 + 법규/조례 챗봇 + 관리자 대시보드 풀스택 웹 애플리케이션

**핵심 기능 3가지**:
1. 평면도 이미지 업로드 → AI가 공간 분석 + 준법 평가 + 자연어 리포트 생성
2. 챗봇에 질문하면 → 조건에 맞는 도면 검색 or 건축 법규/조례 답변
3. 관리자 대시보드 → 사용자/도면/채팅 통계 + 도면 DB 관리

## 슬라이드 4: CV 파이프라인 (평면도 분석)

### 4단계 순차 추론 파이프라인
```
입력: 평면도 이미지 (PNG)
         │
    ┌────▼─────┐
    │ 1. OBJ   │  YOLOv5 객체 탐지
    │          │  → 변기, 세면대, 싱크대, 욕조, 가스레인지 (5개 클래스)
    └────┬─────┘
    ┌────▼─────┐
    │ 2. OCR   │  YOLOv5 + CRNN 텍스트 인식
    │          │  → 공간 라벨 탐지 (거실, 침실, 주방 등)
    └────┬─────┘
    ┌────▼─────┐
    │ 3. STR   │  FCN 세그먼테이션
    │          │  → 출입문, 창호, 벽체 (3개 클래스)
    └────┬─────┘
    ┌────▼─────┐
    │ 4. SPA   │  DeepLabV3 세그먼테이션
    │          │  → 13개 공간 분류 (거실, 침실, 주방, 발코니 등)
    └────┬─────┘
         │
    ┌────▼──────────┐
    │ 결과 통합      │  ResultAggregator
    │ (Aggregator)  │  → 공간별 포함 객체 매핑
    │               │  → 인접 관계(edge) 생성
    │               │  → topology_graph.json
    └────┬──────────┘
         │
    ┌────▼──────────┐
    │ RAG LLM 분석  │  GPT-5.2 + 사내 평가기준(pgvector 검색)
    │               │  → 구조화된 분석 리포트 (FloorPlanAnalysis)
    │               │  → 13개 정량 메트릭 추출
    │               │  → 준법 등급 평가
    └───────────────┘
```

### 인식하는 23개 카테고리
- **공간 (13종)**: 거실, 침실, 주방, 현관, 발코니, 화장실, 다용도실, 드레스룸, 팬트리, 엘리베이터홀, 계단실, 다목적공간, 기타공간
- **객체 (5종)**: 변기, 세면대, 싱크대, 욕조, 가스레인지
- **구조 (3종)**: 출입문, 창호, 벽체

## 슬라이드 5: 13개 정량 메트릭

평면도 분석 시 추출하는 13개 지표:

| # | 메트릭 | 타입 | 설명 |
|---|--------|------|------|
| 1 | structure_type | 문자열 | 판상형 / 타워형 / 혼합형 |
| 2 | bay_count | 정수 | 베이 수 (채광면 개수) |
| 3 | room_count | 정수 | 방 개수 |
| 4 | bathroom_count | 정수 | 화장실 개수 |
| 5 | windowless_count | 정수 | 무창 공간 수 |
| 6 | balcony_ratio | 실수 | 발코니 면적 비율 (%) |
| 7 | living_room_ratio | 실수 | 거실 면적 비율 (%) |
| 8 | bathroom_ratio | 실수 | 화장실 면적 비율 (%) |
| 9 | kitchen_ratio | 실수 | 주방 면적 비율 (%) |
| 10 | compliance_grade | 문자열 | 준법 등급 (최우수~불합격) |
| 11 | ventilation_quality | 문자열 | 환기 품질 (우수~미흡) |
| 12 | has_special_space | 불리언 | 특화공간 유무 (드레스룸, 팬트리 등) |
| 13 | has_etc_space | 불리언 | 기타공간 유무 |

### 준법 평가 기준 4가지
- **채광**: 4Bay=최우수, 3Bay=우수, 2Bay=보통 / 무창비율 30% 이하 권장
- **환기**: 맞통풍 구조 필수, 주방·욕실 환기 확보
- **가족융화**: LDK(거실+식당+주방) 30~40% 비율 적정
- **수납**: 10% 이상=우수, 5~10%=보통, 5% 미만=부족

## 슬라이드 7: 도면 검색 RAG (Hybrid RAG)

### ArchitecturalHybridRAG 파이프라인

**텍스트 검색 모드**:
```
질문: "방 3개, 화장실 2개인 판상형 도면 찾아줘"
         │
    ┌────▼─────────────┐
    │ 의도 파악         │  메타데이터 추출 (방수, 구조타입 등)
    └────┬─────────────┘
    ┌────▼─────────────┐
    │ 하이브리드 검색   │  벡터 유사도 (0.8) + 텍스트 검색 (0.2)
    │                   │  + 메트릭 필터 (구조, 방수, 베이, 화장실)
    └────┬─────────────┘
    ┌────▼─────────────┐
    │ LLM 답변 생성    │  [도면 #N] 마커 포함 구조화 답변
    │                   │  + 도면 선택 근거 + 공간 설명
    └──────────────────┘
```

**이미지 유사 검색 모드**:
```
이미지 업로드
         │
    ┌────▼─────────────┐
    │ CV 분석           │  13개 메트릭 + 1024-dim 임베딩 생성
    └────┬─────────────┘
    ┌────▼─────────────┐
    │ pgvector 검색     │  코사인 유사도 + 이산형 필터
    │                   │  (Progressive Filter Relaxation)
    └────┬─────────────┘
    ┌────▼─────────────┐
    │ LLM 비교 설명     │  [유사 도면 #N] 마커 포함
    │                   │  유사점/차이점 구조화 설명
    └──────────────────┘
```

## 슬라이드 8: 법규/조례 RAG (ChatbotService_v2)

### 5단계 파이프라인
```
질문: "서울시 강남구 역삼동에서 단독주택 지을 수 있나요?"
         │
    ┌────▼───────────────┐
    │ 1. 정규화           │  주소/질문 정제
    └────┬───────────────┘
    ┌────▼───────────────┐
    │ 2. 케이스 분류      │  CASE1: 주소만
    │                     │  CASE2: 주소 + 용도지역
    │                     │  CASE3: 용도지역/활동/법조문
    └────┬───────────────┘
    ┌────▼───────────────┐
    │ 3. DB 검색          │  land_char (토지특성)
    │                     │  law (법규조례)
    │                     │  useBuilding (건축물용도)
    │                     │  + 벡터 유사도 검색
    └────┬───────────────┘
    ┌────▼───────────────┐
    │ 4. 특수 쿼리 감지   │  건폐율, 용적률, 높이제한
    │                     │  건축허가, 법 비교
    │                     │  → ZONE_REGULATIONS 사전 활용
    └────┬───────────────┘
    ┌────▼───────────────┐
    │ 5. LLM 답변 생성   │  GPT-4o-mini + Cross-encoder 리랭킹
    └────────────────────┘
```

### 데이터 규모
| DB 테이블 | 내용 | 규모 |
|----------|------|------|
| land_char | 토지특성정보 (주소, 용도지역, 지목, 면적) | CSV 대량 데이터 |
| law | 법규조례 (지역별 허가/불가/조건부) | CSV 대량 데이터 |
| useBuilding | 건축물용도 정의 (카테고리, 시설명, 설명) | CSV 데이터 |
| internal_eval | 사내 평가 기준 문서 + 1024-dim 임베딩 | JSON 데이터 |

## 슬라이드 16: 핵심 기술 포인트 요약

### 1. CV 파이프라인
- **4종 딥러닝 모델** 순차 추론 (OBJ → OCR → STR → SPA)
- **23개 카테고리** 인식 (13 공간 + 5 객체 + 3 구조)
- **토폴로지 그래프** 자동 생성 (공간 노드 + 연결 엣지)

### 2. RAG 시스템
- **하이브리드 검색**: 벡터 유사도(0.8) + 텍스트 매칭(0.2)
- **Progressive Filter Relaxation**: 검색 결과 부족시 자동 필터 완화
- **Cross-encoder 리랭킹**: 검색 정확도 향상

### 3. 에이전트 아키텍처
- **오케스트레이터**: 입력 타입/의도 자동 분류 → 적절한 에이전트 라우팅
- **폴백 메커니즘**: 1차 에이전트 실패시 자동 전환
- **구조화 출력**: LLM structured output으로 일관된 JSON 응답

### 4. 마커 기반 답변 시스템
- `[도면 #N]` / `[유사 도면 #N]` 마커로 도면 ID 임베딩
- 프론트엔드에서 파싱 → 썸네일 + 구조화 설명 렌더링
- 백엔드에서 floorplan_ids → S3 이미지 URL 자동 변환

### 5. 보안 & 인프라
- **JWT 인증**: rememberMe 분기(24h/1h) + 자동 갱신
- **이메일 인증**: 6자리 OTP (5분 만료, 30초 쓰로틀)
- **AWS S3**: 도면 이미지 클라우드 저장
- **pgvector**: 1024-dim 벡터 유사도 검색
